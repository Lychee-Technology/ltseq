# LTSeq 源码级白话解读：顺序驱动的数据工厂

## 开场
想象一个不可能完成的任务：你要把全国所有交易记录按时间顺序排好，然后找出每个人连续三天消费上升的那段时期，还要顺便算出每一段的起点、终点和幅度。数据大得像一座城，时间紧得像一趟高铁。你需要的不是“更快的算盘”，而是一个懂“顺序”的工厂。

## 主比喻
LTSeq 就是这样一座“时序工厂”。它不是把数据当成一堆散乱零件，而是把它们当作一条不断流动的队伍。你可以把它想成一条有规矩的生产线：每一条数据进来都要排队，排好队才允许做“前后对比、连续统计、滚动计算”这种依赖顺序的工作。

## 拆黑盒
这座工厂有两层员工。一层是“前台接待员”，住在 Python 这层。他的工作是接订单、记笔记、画流程图，绝不亲手加工。另一层是“车间主任”，住在 Rust 这层。他才是动手的人，真正把数据推进机器里、把流水线跑起来。

前台接待员是个非常会“读心”的人。他会把你写的那句“筛一下年龄大于 18 的人”翻译成一张可执行的“加工单”。在源码里，这件事是靠一个“角色扮演”完成的：你以为自己在写一个普通的函数，其实那个参数根本不是数据，而是一个“假人模特”。你对它说“取年龄”“大于 18”，它就把这些动作记录成一棵“表达式树”。这就好比你用手势给厨师比划“要切丝、要加盐”，厨师记下步骤，真正下锅要等到后面。

这就是 LTSeq 的懒执行。它不着急算，而是先把你的每一步都记成操作清单。等你真正说“给我看结果”，才一起开机器。这么做是为了省时间和省钱，因为连着几步一起执行，能避免重复劳动。这在源码里表现得很直白：很多操作只是在更新“计划”，只有在 show、to_pandas、count 这种“终端动作”出现时，才真正去拉数据。

现在，数据真正进入车间。车间主任不是凭空造机器，而是请来了一个老练的工厂总管：DataFusion。你可以把它理解为一条成熟的装配线，专门做批量处理。它最擅长把数据切成一批一批整齐的“箱子”，一次装配一整批，速度快、体力省。在源码里，这个“装配线”由 SessionContext 负责配置，批量大小、并行工人数量都提前定好，相当于把工厂的班次和流水线速度调到适合的档位。

LTSeq 的核心特点是“顺序必须真实存在”。这不是说它喜欢慢吞吞，而是它非常在意“谁先谁后”。你要做“上一行、下一行、过去五天均值”这种动作，必须先告诉它排序规则。这就像你要统计“队伍里每个人前面第三个人是谁”，你必须先让队伍排好，否则这事没有意义。源码里，这条规矩是硬性规定：没有明确排序，窗口类操作直接报错。这里的“窗口”，可以理解成一个滑动的小尺子，总是贴着队伍走，量“当前人附近几位”的情况。

说到窗口，LTSeq 的做法很“实在”。它会把当前表先收集成一摞临时表（就像把零件全部摆到临时工作台），然后用 SQL 的“窗口语句”去加工。这是因为窗口操作很复杂，用通用流水线最靠谱。在源码里，窗口操作会把数据暂存到内存表，再执行一条完整的查询语句。这不是为了绕远路，而是为了确保顺序计算正确，而且能享受 DataFusion 的优化。

接下来是连接两张表的场景。你可以把连接理解为“把两条队伍按身份证号配对站队”。但是现实中有个麻烦：如果两边都有人叫“名字”，管理系统会弄混谁是谁。LTSeq 的解决办法很像工厂贴标签：右边队伍的所有人的名字都先加上前缀，变成“右边_名字”，这样就不会撞名。源码里正是这么做的：先把右表列名全部临时改名，连好队再改回你看得懂的名字。

除了传统连接，LTSeq 还有一种更“轻量”的方式，叫链接。你可以把它理解为“给人挂一张指路卡片”，卡片上写着“去那边第几排第几人”。它不会立刻把两条队伍合成一条，而是先记一条指路关系。只有当你真的去取对方信息时，它才把队伍合并。这种做法的好处是省成本：很多时候你只需要左边队伍的信息，没必要搬动右边队伍。源码里，这种设计表现为“链接表”先保持虚拟形态，只有碰到对方列时才触发物化。

还有一种特别适合时间序列的对齐方式：按时间找“最近的那条”。这就像你要给每一笔交易找最近一次报价。LTSeq 的做法不是傻乎乎逐条比对，而是用“查目录”的方式快速定位：右边队伍按时间排好，左边每个人拿着时间戳去做二分查找。源码里就是经典的二分搜索逻辑，向前找最近、向后找最近、或者找最近的那个，速度比全表扫描快得多。

## 数据旅程
当数据在工厂里走一圈，会发生什么？想象一条交易记录进门：前台接待员先给它画好流程图，车间主任按流程把它塞进装配线。它先被排序站队，然后滑动窗口的尺子贴着它走，算出“前一笔、后一笔、过去五笔均值”。接着它可能被“链接卡片”指向商品表，但只有当你真的要求商品名称时，才去合并。最后它被展示出来，或者被转成你熟悉的表格格式。整个过程里，数据从未“随意散开”，它一直是一条队伍，只是不断地被贴标签、被计算、被抽取。

## 一句话价值
一句话总结：LTSeq 是一座懂顺序的时序工厂，让数据像队伍一样排好队，所有“前后关系”的计算才有意义。
